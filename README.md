# AI SQL Agent (RAG + Text-to-SQL)

ИИ-агент на базе **LangGraph** и векторной базы знаний **Qdrant**, который автоматически переводит текстовые вопросы пользователя на естественном языке в точные аналитические SQL-запросы к реляционной базе данных.

## 🚀 Что делает агент?

1. **Контекстный поиск схем (RAG):** При получении вопроса пользователя (например, *«Покажи email клиентов с максимальными тратами»*), агент выполняет локальный гибридный поиск (Dense + Sparse) в коллекции `db_metadata` базы данных Qdrant. Он извлекает точные структуры необходимых таблиц, описания полей и примеры валидных SQL-запросов (Few-shot context).
2. **Генерация SQL (Text-to-SQL):** На основе извлеченного контекста языковая модель (LLM) строит логику рассуждения и генерирует строго структурированный, синтаксически верный SQL-запрос.


## 🛠 Технологический стек

* **Оркестрация агента:** LangGraph (управление состояниями и циклическими графами).
* **Векторная база данных:** Qdrant (хранение метаданных).
* **Локальные эмбеддинги:** FastEmbed (гибридный поиск: плотные векторы `MiniLM` + разреженные векторы `SPLADE` для точного совпадения ключевых слов).
* **Языковые модели (LLM):** Подключение любых моделей (OpenAI, DeepSeek, Anthropic) через OpenRouter API и LangChain.
* **Менеджер пакетов:** `uv` (для установки зависимостей).

## Как запустить

### Локальный запуск qdrant и postgres.
docker compose up -d

http://localhost:6333/dashboard

### Создание схемы и загрузка тестовых документов:
uv run prepare_qdrant_data.py

### Запуск графа
uv run agent.py




Этапы разработки
1) RAG-> SQL_GEN
2) RAG -> SQL_GEN -> PG
3) RAG -> SQL_GEN -> PG -> Answear 
4) RAG -> SQL_GEN -> PG -> Answear (with Memory)
-5) Agentic Loop
